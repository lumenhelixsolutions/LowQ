# Contributing to LowQ

LowQ welcomes noncommercial research, experiments, issue reports, benchmark traces, simulator profiles, and adapters.

## Contribution principles

- Function first.
- Claims require evidence.
- No hardware heat claims without telemetry.
- Every optimization must be benchmarkable.
- Prefer simple core logic and replaceable adapters.
- Preserve reversibility where transformations alter workload content.

## What to contribute

- benchmark workloads,
- telemetry import templates,
- simulator import examples,
- report templates,
- adapter prototypes,
- proof certificate validators,
- documentation improvements,
- reproducibility notes.

## Commercial contributions
If your contribution is intended for commercial deployment or customer work, discuss licensing before integrating it.
