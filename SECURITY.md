# Security Policy

LowQ Phase 1 is a browser-first, local-first proof stack.

## Current security posture

- The lab runs locally in the browser.
- No network calls are required.
- No API keys are required.
- Telemetry imports are file-based.
- Exported certificates are local files.

## Future security requirements

Before production deployment, LowQ Bridge and LowQ Engine require:

- threat model,
- dependency review,
- telemetry data minimization,
- signed release artifacts,
- least-privilege sensor access,
- audit logging,
- secure update path,
- enterprise data handling policy.

Report issues through the repository issue tracker until a formal security contact is established.
